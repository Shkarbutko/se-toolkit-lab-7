The bot will be developed in small, testable steps. First, I will create a clean project scaffold inside the bot directory. The entry point will be bot.py, and command logic will be placed in separate handler modules. This keeps Telegram-specific code away from business logic, so handlers can be tested without connecting to Telegram. The first version will support offline test mode with the --test argument. This mode will route a text command to the same handlers that the Telegram bot will later use.

Next, I will add backend integration through a services layer. The services directory will contain clients for the LMS/backend API and for the LLM API. Later tasks will replace placeholder responses with real data. Intent routing will be added gradually: simple slash commands first, then natural language queries. Deployment will happen on the VM after each task. I will verify every change with uv sync, offline test commands, and then Telegram runtime checks.
